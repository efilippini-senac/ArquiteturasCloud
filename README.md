# Simple Orders – Evolução de Arquiteturas

Este repositório é usado em laboratório didático para demonstrar a evolução de arquiteturas de aplicações em nuvem.

A MESMA aplicação é implementada em diferentes estilos arquiteturais.

## Objetivos educacionais

- Entender o que é um monolito
- Evoluir para um monolito distribuído
- Preparar base para microserviços e event-driven
- Relacionar arquitetura com alta disponibilidade e cloud

## Fases disponíveis

- Fase 1 – Monolito (tudo em uma VM)
- Fase 2 – Monolito distribuído (frontend, backend e banco separados)
